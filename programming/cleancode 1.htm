Clean code manifesto

1-Meaningful names ovr short names 2-Meaningful code over clever code 3-Using small functions or modules over big blocks of code 4-Using a model class as parameter of a function over having long list of parameters for it; 5-Avoid using null and nullable. 6-Using private fields and methods over using publics. 7-Using generics over repeating your code 8-least needed functionlity for classes over comprehensive classes 9-Single responsibility of functions and classes over multipurpose functions and classes 10-using local variables and parameters over global variables 11-Using constants instead of magic numbers/magic strings 12-Using ready made libraries and components over Inventing the wheele 13-Least coupling and maximum cohesion of classes.
